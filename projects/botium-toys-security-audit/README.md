# Botium Toys Security Audit

## Overview

This project documents a cybersecurity audit conducted for Botium Toys, a fictional retail company. The objective was to assess the organization's security posture, identify risks, evaluate existing controls, and review compliance requirements.

## Audit Scope

The audit covered the entire security program of Botium Toys, including:

* Employee devices and equipment
* Internal network infrastructure
* Internet connectivity
* Data storage and retention systems
* E-commerce and inventory management systems
* Accounting and telecommunications systems
* Physical business assets
* Legacy systems requiring maintenance

## Audit Goals

* Identify organizational assets
* Evaluate existing security controls
* Assess compliance requirements
* Identify security gaps and risks
* Recommend improvements to strengthen security

## Assets Reviewed

### Hardware Assets

* Desktop and laptop computers
* Smartphones
* Remote workstations
* Headsets and peripherals
* Surveillance cameras

### Business Assets

* Retail inventory
* Warehouse operations
* Storefront systems

### Information Systems

* Accounting systems

* Inventory management systems

### Infrastructure

* Internet access
* Internal network
* Data storage systems
* Legacy systems

## Risk Assessment

### Risk Score

**8/10 (High Risk)**

### Risk Summary

The organization lacks several critical security controls and demonstrates weak asset management practices. These deficiencies increase the likelihood of unauthorized access, data exposure, operational disruption, and regulatory non-compliance.

## Key Findings

### Security Weaknesses

* No implementation of least privilege access controls
* No separation of duties
* No encryption for cardholder data
* No intrusion detection system (IDS)
* No backup strategy
* No disaster recovery plan
* Weak password policy
* No centralized password management solution

### Existing Controls

* Firewall protection
* Antivirus software
* Data integrity controls
* Physical security locks
* CCTV surveillance
* Fire detection and prevention systems
* GDPR breach notification procedures

## Compliance Review

### PCI DSS

Several requirements were not met, including:

* Restricted access to cardholder data
* Encryption of payment information
* Secure password management

### GDPR

Strengths:

* 72-hour breach notification process
* Privacy policies and procedures

Areas for improvement:

* Data classification
* Data inventory management
* Protection of customer information

### SOC Controls

Strengths:

* Data integrity
* Data availability

Areas for improvement:

* User access management
* Confidentiality of sensitive information

## Recommendations

1. Implement least privilege access controls
2. Deploy encryption for sensitive information
3. Establish a centralized password management solution
4. Implement backup and recovery procedures
5. Deploy an intrusion detection system
6. Improve asset inventory and classification processes
7. Strengthen password complexity requirements
