# Incident Response - DoS Attack Analysis using NIST CSF

## 📌 Project Overview

This project presents an incident response report based on a simulated Denial of Service (DoS) attack against a company's internal network. The analysis follows the NIST Cybersecurity Framework (CSF), demonstrating how security incidents can be identified, analyzed, mitigated, and prevented.

This project was developed as part of my Cybersecurity studies and is included in my portfolio to demonstrate practical knowledge of incident analysis and network security.

---

## 🎯 Scenario

A multimedia company experienced a Denial of Service (DoS) attack that disrupted its internal network for approximately two hours.

The investigation revealed that an attacker exploited a misconfigured firewall to launch an ICMP Flood attack, overwhelming the network infrastructure and preventing legitimate traffic from accessing internal resources.

## 📖 NIST CSF Functions Applied

### Identify
- Identified the vulnerable firewall configuration.
- Assessed risks affecting network availability.

### Protect
- Implemented ICMP rate limiting.
- Configured source IP validation.
- Improved firewall hardening.

### Detect
- Deployed network monitoring.
- Added IDS/IPS capabilities.
- Established traffic anomaly detection.

### Respond
- Blocked malicious ICMP traffic.
- Restored critical services.
- Investigated the root cause.

### Recover
- Restored affected systems.
- Implemented long-term security improvements.
- Updated monitoring and recovery procedures.
